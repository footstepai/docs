---
title: 'search_along_route'
description: 'Find places along a route ranked by detour time'
---

Calculate a route and search for places along the corridor. Results are ranked by how little detour they add to your journey. Useful for finding stops on the way to a destination without going too far off course.

You can set the maximum acceptable detour (1 minute to 1 hour) to control how far off-route results can be.

## Example prompts

- "Find coffee shops on my way from Kings Cross to Cambridge"
- "Are there any petrol stations along the route that won't add more than 5 minutes?"
- "Show me restaurants between here and the airport with minimal detour"
- "What supermarkets can I stop at on my drive to Oxford?"

## What you get back

The original route plus a list of places found along the corridor. Each place includes the detour it would add (in seconds and meters), and where along the route it sits (as a percentage from start to end). Results are ranked by detour time so the most convenient stops come first.

## Rendering on a map: `include_geometry`

By default this tool returns a lean response — the encoded polyline of the route and the sample-by-sample elevation profile are stripped out so standalone LLM agents (which can't render maps) don't pay for them in tokens.

If your client renders results on a map (Mapbox, MapLibre, Leaflet, etc.), pass `include_geometry: true`. The response then carries the route polyline and elevation profile per leg alongside the same summary fields (and the candidate places, which are unaffected). The map renders the geometry; the LLM still reasons over the summary.

Setting `include_geometry: true` on every call from a map-rendering client is tedious. See the [getting-started guide](/mcp/getting-started#map-rendering-clients) for a one-line Vercel AI SDK middleware that injects the flag automatically.
